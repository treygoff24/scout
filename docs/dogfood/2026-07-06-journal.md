# Scout dogfood journal — 2026-07-06

Agent-perspective notes from first real usage (Claude/opus session, reviewing Codex's overnight build with scout itself). Raw, chronological. Tags: BUG / FRICTION / IDEA / GOOD / TOKENS.

---

- **BUG**: `scout --version` doesn't exist — the arg falls through to the query path, so you get an `index_stale` envelope (exit 11) for the "query" `--version`. Any unrecognized first arg is silently treated as a query. An agent probing capabilities the standard CLI way gets a confusing envelope instead of a version string. Fix: real `--version`/`-V`, and consider warning when a query looks like a flag (starts with `-`).
- **GOOD**: `--help` is 5 lines. Perfect token economy for an agent. Whole surface visible at a glance.
- **GOOD**: index_stale envelope carried `hint: "run scout index"` and `reason: content_hash_mismatch` — self-healing instructions in-band. That's exactly right for agents.

## Index + brief

- **GOOD**: `scout index` on this repo: 15 cards, 1.0s, $0.016 actual. Caught a real `.env` inside `.delegate/runs/.../scratch/` and denied it at the walk — the sensitive boundary works live, first try.
- **FRICTION**: it walked into `.delegate/` (agent-tooling scratch) at all. Harness litter dirs (`.delegate`, `.claude`, session scratch) should probably be default-excluded like VCS dirs — they're noise cards and a place secrets accumulate.
- **FRICTION**: `estimated_usd` 0.134 vs actual 0.016 — estimate ~8x over actual on a warm/incremental index. Overestimating is the safe direction, but 8x makes an agent's cost-benefit math useless.
- **GOOD**: `brief` is ~11.8KB (~3K tokens) — a reasonable session-start hydration price. `model_hint: true` tagging on role fields is exactly the trust-separation the plan demanded, visible in-band.
- **FRICTION**: brief lists `.gitignore` as an "entry point" with a vacuous model hint ("Defines files ignored by Git"). Entry-point ranking treats harness/config litter as peers of src/main.rs. `harness_meta: true` exists but only CLAUDE.md got it.

## Query loop

- **GOOD**: queries cost $0.07–0.09, return in 1–2s, and line anchors are precise (redact_outbound at 601/830, walk deny at 1453, atomic pointer rename at 1868). As a map for a code review this beat grepping cold — I went straight to the five load-bearing sites.
- **GOOD**: cost estimates print to stderr BEFORE spend (router and extractor separately). Non-negotiable honored.
- **BUG?**: `state: "partial"` returned with `errors: []` and nothing in the envelope saying WHAT was partial. Agent can't distinguish "some chunks failed" from "findings were dropped" from cosmic rays. If dropped>0 triggers partial, that's wrong per the design (dropped findings are normal firewall operation, and the same run had ok-state siblings with 0 dropped... no wait, the 23-finding run was state ok with 0 dropped — hypothesis: dropped>0 ⇒ partial. Needs source check.)
- **SYSTEMATIC FALSE-DROP**: gemma strips markdown formatting (`**bold**`, backticks) when copying quotes from .md files → `quote_not_in_file` drops of REAL facts. Saw it 5 times in 4 queries, always on design-doc lines with inline markdown. Options: (a) add a markdown-syntax-insensitive normalization tier to the firewall matcher (still mechanical, still verbatim-modulo-markup), (b) strengthen the extractor prompt ("copy formatting characters exactly"). (a) is more durable — you can't prompt gemma into perfection.
- **FRICTION**: `.scout/last-run.json` is a single slot — my 2nd query silently clobbered the 1st's findings. Agents naturally run query bursts. Per-query artifacts exist for eval runs (`.scout/eval-runs/...`) but not normal queries. Idea: `.scout/runs/<ts>.json` ring buffer (keep last N), `last-run.json` stays as the convenience pointer.
- **FRICTION**: last-run.json is NOT the full envelope — just `{candidate_files, dropped, findings, query}`. No state, no spend, no timings. The design says it's the hydration re-read artifact; losing spend/state there breaks post-hoc accounting.
- **BUG**: `scout --version` → falls through to query path, exit 11, `index_stale` envelope. Any first arg starting with `-` should be rejected as unknown-flag, not queried.
- **TOKENS**: a raw query envelope is ~6–10KB. Fine for one query; for a burst of 5 an agent eats 30-50KB of transcript. Idea: `--compact` mode that drops `deterministic_score`/`router_rank` and truncates `dropped` to counts-by-reason.

## Staleness, negatives, edges

- **BIG FRICTION**: editing ONE file (this journal!) flipped the whole index to `index_stale` and queries hard-refuse (exit 11) until re-index. The agent's primary loop is edit→query→edit; every edit costs a round-trip. Mitigations, in order of my preference: (1) auto-incremental-reindex-on-query behind a flag (`--refresh`), since incremental re-index measured $0.003/1s — cheap enough to just do; (2) serve the stale snapshot with `state: ok` + `stale: true` warning field unless files relevant to the candidates changed; (3) at minimum the envelope hint should say "1 file changed since gen-X" so the agent knows the re-index is trivial.
- **GOOD**: incremental re-index after a 1-file change: $0.0033, 15 cards, ~1s. Card reuse works. The slow loop is genuinely cheap when warm.
- **CONFIRMED FOOTGUN**: `scout indx .` (typo) silently becomes query "indx". Unknown-subcommand detection needs an edit-distance guard or at least "did you mean index?" to stderr. Cost of typo today: a wasted API query + a confusing envelope.
- **SUBTLE**: negative probe "Where is the websocket server implemented?" on this repo → `state: ok`, 1 finding: the killtest doc line SAYING websocket negative probes returned zero survivors. Verbatim, honest, correctly cited — and self-referentially poisoned: docs ABOUT testing-for-X match queries FOR X. On any corpus whose docs discuss absent features, "ok + tangential finding" will read as "found it" to a skimming agent. Ideas: `unanswered` should probably fire when max deterministic_score is this weak (0.33 here vs 1.5-1.8 on real hits) — a "confidence floor" on state assignment; or surface score percentile so agents can threshold.
- **GOOD**: `last_run_path` now visible in envelope data — self-describing artifact location.
- **NOTE**: gemma preserved `**zero survivors**` markdown fine here — the markdown-stripping false-drop is intermittent, which fits "sometimes strips formatting." Firewall-side normalization still the durable fix.
- **INCONSISTENCY**: with the index stale, `query` hard-refuses (`index_stale`, exit 11) but `brief` silently serves the old snapshot as `state: ok` with no staleness field. Same staleness, opposite policies, and brief's is silent. Pick one policy (I'd serve-stale-with-warning for both) or at least make brief carry `stale: true`.

## Meta: using scout to review scout

- **VALIDATION**: every line anchor scout gave me (redact at 601/830, patterns at 961, walk deny at 1453, pointer rename at 1868) was independently confirmed by the opus trust reviewer reading the file cold. The map was accurate, not just plausible. For seeding a code review, scout queries beat cold grep: 3 queries (~$0.25, ~6s) gave all three reviewers their starting anchors.
- **FRICTION**: what scout could NOT answer is review-shaped questions ("is the backoff exponential?", "does partial conflate drop reasons?"). It retrieves claims; it can't evaluate them — correct per the never-synthesize design, but worth remembering the division of labor: scout = where to look, reviewer model = what to think. The design doc already says this; usage confirms it viscerally.
- **IDEA**: the killtest-evidence and design docs kept outranking src/main.rs for "how does X work" queries — docs describe intent in query-shaped language, code doesn't. For code-review use, a `--code-only` / adapter filter flag would cut prose echo. (Flip side: for policy corpora the prose bias is exactly right.)
- **IDEA** (from eval audit): eval runs print aggregate metrics to stdout only; artifact dirs have per-query files but no summary.json. Gate numbers should be a stored artifact, not prose transcription.
- **IDEA**: `scout` with no args prints usage and exits 2 with NO envelope. Fine for humans, mildly inconsistent for agents — every other path emits an envelope.

## Post-campaign verification (same day, after waves 1-3)

Every P0/P1 item and 15 UX items shipped across three reviewed waves (f3ba5aa, b2ba48f, d155630 + final micro-batch). Live re-test of each original friction scenario: all fixed — see the scorecard in the session log. Highlights measured live:
- m1 gate post-refactor: recall 100, coverage 80.95 (identical to baseline), poison 0, 0 retries, $0.58.
- m3 gate post-refactor: recall 100, coverage 91.30 (**up from 82.61** — the markdown-normalized tier recovered the false-dropped facts), negatives clean, 0 retries, $0.67.
- edit→query round-trip now one command: `--refresh` reindexed + queried in a single call, spend merged into one envelope, artifact matches stdout.
- index_stale envelope now says `changed_files: 1` — exactly the re-grounding an agent needs to decide instantly.

New notes from the verification pass itself:
- **GOTCHA (meta)**: this journal is part of the corpus, so it self-poisons negative probes — "where is the websocket server" now surfaces the journal's own sentence about websocket negative probes, scored 1.25 (genuinely relevant!). weak_signal correctly doesn't fire. Not a bug — the retina reports what the corpus says — but dogfood/test artifacts living inside the corpus will echo. If this matters later: a `.scoutignore` would be the durable answer.
- **LESSON (process)**: my fold instruction "persisted artifact must equal the printed envelope" collided with "full detail always persists" and Codex resolved toward the newer instruction — compact briefly leaked into artifacts. Caught by re-dogfooding, fixed same hour. Conflicting spec sentences are the coordinator's bug, not the implementer's.
- **CALIBRATION**: deterministic_score scale shifts with card composition (real hits 1.5-1.8 on one generation, 0.93-1.18 after re-carding). Any absolute threshold on it is wrong; weak_signal is now ratio-based (max finding score vs top candidate score, floor 0.4).

Deferred, consciously: tok/min limiter (req/min bucket only), refresh-then-changed race (self-describing now), lexicographic runs-file ordering (safe until year 2286), .pub public-key over-deny reversal outside denied dirs (done), `.scoutignore`.
